    1. Create `apps/mobile/src/stores/collection.ts` (Zustand store following cards.ts pattern):
       ```typescript
       interface CollectionState {
         mode: 'browse' | 'collection' | 'wanted';
         collectionByCardId: Record<string, number>;  // cardId -> quantity
         wantedByCardId: Record<string, 'high' | 'medium' | 'low'>; // cardId -> priority
         progressBySet: Record<string, { owned: number; total: number; setName: string }>;
         collectionLoaded: boolean;
         wantedLoaded: boolean;
         // Actions
         setMode: (mode: 'browse' | 'collection' | 'wanted') => void;
         setCollection: (items: Array<{ cardId: string; quantity: number }>) => void;
         setWanted: (items: Array<{ cardId: string; priority: 'high' | 'medium' | 'low' }>) => void;
         setProgress: (progress: Array<{ setId: string; setName: string; owned: number; total: number }>) => void;
         // Optimistic update actions
         addToCollection: (cardId: string, quantity?: number) => void;
         removeFromCollection: (cardId: string) => void;
         updateQuantity: (cardId: string, quantity: number) => void;
         addToWanted: (cardId: string, priority?: 'high' | 'medium' | 'low') => void;
         removeFromWanted: (cardId: string) => void;
         updatePriority: (cardId: string, priority: 'high' | 'medium' | 'low') => void;
       }
       ```
       - `addToCollection` optimistically increments collectionByCardId[cardId] (or sets to quantity if new)
       - `removeFromCollection` deletes key from collectionByCardId
       - `updateQuantity` sets exact value; if 0, deletes key
       - `addToWanted` sets wantedByCardId[cardId] = priority (default 'medium')
       - `removeFromWanted` deletes key from wantedByCardId
       - `updatePriority` updates wantedByCardId[cardId]

    2. Create `apps/mobile/src/hooks/useCollection.ts`:
       - `useLoadCollection()`: on mount, fetches GET /collection and GET /collection/progress, populates store. Use apiFetch (no skipAuth -- needs auth). Only fetch if not already loaded.
       - `useAddToCollection()`: returns async function that optimistically updates store, then calls POST /collection { cardId, quantity }. On failure, revert store state.
       - `useRemoveFromCollection()`: returns async function, optimistic delete, calls DELETE /collection/:cardId. On failure, revert.
       - `useUpdateQuantity()`: returns async function, optimistic update, calls PUT /collection/:cardId { quantity }. On failure, revert.
       - `useBulkUpdateCollection()`: returns async function that calls POST /collection/bulk { setId, additions, removals }, then refreshes full collection data on success (no optimistic -- wait for server).
       - `useRefreshProgress()`: returns async function that re-fetches GET /collection/progress and updates store.
       - Debounce rapid addToCollection calls (300ms) -- queue rapid increments into single API call.

    3. Create `apps/mobile/src/hooks/useWanted.ts`:
       - `useLoadWanted()`: on mount, fetches GET /wanted, populates store. Only fetch if not already loaded.
       - `useAddToWanted()`: optimistic update + POST /wanted { cardId, priority }
       - `useRemoveFromWanted()`: optimistic delete + DELETE /wanted/:cardId
       - `useUpdatePriority()`: optimistic update + PUT /wanted/:cardId { priority }

    All hooks follow error handling pattern: try/catch, revert optimistic state on error, surface error via return value or toast.
