# Phase 13: Design System Foundation - Context

**Gathered:** 2026-03-21
**Status:** Ready for planning

<domain>
## Phase Boundary

Create a single source of visual truth: a shared token package in `packages/shared/tokens/` exporting colors, typography, spacing, elevation, and motion curve values. All color references migrate to semantic aliases. A backward-compatible shim keeps existing screens working. Web CSS custom properties are generated from the same TS source. A hardcoded value audit produces a migration tracking list for future phases.

</domain>

<decisions>
## Implementation Decisions

### Color Palette & Semantics
- **D-01:** Design a new unified dark palette that works on both mobile and web — neither current mobile (#0f0f1a) nor web (#0a0a0a) is canonical. The gold accent (#f0c040) stays.
- **D-02:** Core semantic color set: surface, onSurface, accent, error, success, warning. No extended Material Design 3 aliases.
- **D-03:** Rarity-specific colors (diamond blue, star gold, crown purple) included as first-class tokens — they appear in 10+ files with hardcoded hex today.

### Token Hierarchy & Structure
- **D-04:** Two-tier hierarchy: Primitive (raw values like blue-900, gold-500) → Semantic (surface, accent, error). No component-level tokens in this phase.
- **D-05:** Named spacing sizes: xs/sm/md/lg/xl/xxl (matching current theme.ts). Add 2xs and 3xl if needed.
- **D-06:** Elevation tokens included: 3-4 levels (none, low, medium, high) as shadow values.
- **D-07:** Motion curve and duration tokens defined in the token package. Phase 15 builds animation hooks that consume these values.

### Migration & Backward Compatibility
- **D-08:** Re-export shim — `theme.ts` becomes a thin layer mapping old names (colors.background, spacing.md) to new token imports. Zero changes needed in the 60 consuming files.
- **D-09:** Hardcoded value audit produces a tracking spreadsheet: file, line, hardcoded value, suggested token replacement. Referenced during Phases 16/17.
- **D-10:** Web `constants.ts` THEME object replaced directly (only 1 consumer file + globals.css — small blast radius, no shim needed).
- **D-11:** Actual file-by-file migration of hardcoded values happens in Phases 16/17 as each screen is touched. Phase 13 creates tokens + shim + audit list only.

### Package Layout & Cross-Platform Sync
- **D-12:** Token package lives at `packages/shared/tokens/` — subdirectory of existing shared package. Both apps already depend on @pocket-trade-hub/shared.
- **D-13:** Web CSS custom properties generated by a Node build script that reads TS token values and outputs `tokens.css`. Runs as a Turborepo pipeline step before web build.
- **D-14:** Token package exports raw values only (plain objects: colors, spacing, typography, elevation, motion). Each platform wraps them as needed.
- **D-15:** Generated `tokens.css` is a separate file imported in `globals.css` — keeps generated and hand-written CSS separate.

### Claude's Discretion
- Warning color selection (orange-adjacent to differentiate from gold accent)
- Exact unified dark palette values (within the dark + blue-undertone + gold accent brand)
- Elevation shadow exact values per level
- Motion curve easing functions and duration scale
- Audit spreadsheet format and tooling

</decisions>

## Existing Code Insights

### Reusable Assets
- `apps/mobile/src/constants/theme.ts`: Well-structured token file with colors (13), typography (5 styles), spacing (6 sizes), borderRadius (5 levels) — good base to evolve into the new token package
- `packages/shared/`: Existing shared package with Zod schemas, workspace already configured — tokens subdirectory fits naturally

### Established Patterns
- Mobile: `StyleSheet.create` with theme imports — no framework migration needed, just swap import source
- Web: Tailwind v4 with CSS-first @theme config — generated tokens.css replaces @theme variables
- Monorepo: Turborepo pipeline with pnpm workspaces — build script fits existing infra

### Integration Points
- 60 mobile files import from `constants/theme` — shim intercepts these
- 1 web file imports THEME from `lib/constants` — direct replacement
- `globals.css` @theme block — replaced by tokens.css import
- `packages/shared/src/index.ts` — may need to re-export token package
