  <action>
    1. Install @fastify/cookie in apps/api: `cd apps/api && pnpm add @fastify/cookie`

    2. Update apps/api/src/server.ts:
       - Import and register @fastify/cookie BEFORE authPlugin
       - Update CORS config to support both mobile (any origin) and web (specific origin with credentials):
       ```typescript
       await app.register(cors, {
         origin: (origin, cb) => { cb(null, true); },
         credentials: true,
       });
       ```

    3. Update apps/api/src/plugins/auth.ts:
       - Update authenticate decorator: try request.jwtVerify() first (Bearer header for mobile). On catch, check request.cookies?.accessToken. If found, verify it with fastify.jwt.verify(token) and set request.user = decoded. If neither works, return 401.

    4. Update apps/api/src/routes/auth.ts:
       - After issuing tokens on /auth/signup and /auth/login, ALSO set HTTP-only cookies:
         ```typescript
         const cookieOpts = {
           httpOnly: true,
           secure: process.env.NODE_ENV === 'production',
           sameSite: process.env.NODE_ENV === 'production' ? 'none' as const : 'lax' as const,
           path: '/',
         };
         reply
           .setCookie('accessToken', tokens.accessToken, { ...cookieOpts, maxAge: 15 * 60 })
           .setCookie('refreshToken', tokens.refreshToken, { ...cookieOpts, maxAge: 7 * 24 * 60 * 60 });
         ```
       - Update /auth/refresh: read refreshToken from body (mobile) OR request.cookies?.refreshToken (web). After issuing new tokens, set cookies again.
       - Update /auth/logout: clear cookies with `reply.clearCookie('accessToken', { path: '/' }).clearCookie('refreshToken', { path: '/' })`
       - These cookie operations are harmless for mobile clients (mobile ignores Set-Cookie headers).

    5. Verify existing API tests still pass -- cookie additions should not break Bearer auth flow.
  </action>

    <automated>cd apps/api && pnpm test 2>&1 | tail -20</automated>
  </verify>
