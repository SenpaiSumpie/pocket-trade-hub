    1. In schema.ts, add postTypeEnum ('offering', 'seeking'), postStatusEnum ('active', 'closed', 'auto_closed'), and tradePosts table with: id (text PK), userId (FK to users), type (postTypeEnum), status (postStatusEnum default 'active'), cards (jsonb), createdAt, updatedAt. Add indexes: user_id, type+status composite, created_at, GIN index on cards with jsonb_path_ops.

    2. In packages/shared/src/schemas/post.ts, create Zod schemas:
       - postCardSchema: { cardId: string, language: string, name: string, imageUrl: string, rarity: string nullable, setId: string optional }
       - createPostSchema: { type: enum('offering','seeking'), cards: array(postCardSchema).min(1).max(1) }
       - tradePostSchema: full post shape with id, userId, type, status, cards, timestamps
       - Export types: PostCard, CreatePostInput, TradePost, PostType, PostStatus
       - Export the shared package barrel (update index.ts if needed)

    3. In post.service.ts, implement:
       - createPost(db, userId, input): Check active post count for non-premium users (limit 15). Generate UUID id, insert into tradePosts, return post.
       - getPosts(db, userId, filters): Query tradePosts with status='active', apply filters (type, language via JSONB @> containment, rarity via JSONB, card name via JSONB text search, setId via JSONB). Cursor-based pagination on createdAt. For each post, compute isRelevant: if post type='offering', check if any post card matches user's wanted list (cardId+language); if type='seeking', check if any post card matches user's collection. Sort by newest (default) or relevant-first. Premium users' posts get a 30-minute recency boost.
       - getMyPosts(db, userId): Return all posts for user (any status), ordered by createdAt desc.
       - closePost(db, userId, postId): Set status='closed' where owner matches, return 404/403 if not found/not owner.
       - deletePost(db, userId, postId): Hard delete where owner matches.
       - getActivePostCount(db, userId): Count active posts for premium limit check.
