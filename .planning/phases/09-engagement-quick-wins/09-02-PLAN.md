  <action>
    **1. DB Schema** -- Add to `apps/api/src/db/schema.ts`:
    - `promoCodes` table: id (text PK), code (varchar 30, unique), description (text nullable), premiumDays (integer), maxRedemptions (integer nullable), currentRedemptions (integer default 0), isActive (boolean default true), expiresAt (timestamp nullable), createdAt (timestamp defaultNow). Add unique index on code.
    - `promoRedemptions` table: id (text PK), promoCodeId (text FK to promoCodes), userId (text FK to users), premiumDaysGranted (integer), redeemedAt (timestamp defaultNow). Add unique index on (userId, promoCodeId) and index on userId.

    **2. Shared Schemas** -- Create `packages/shared/src/schemas/promo.ts`:
    - `redeemCodeSchema`: z.object({ code: z.string().min(1).max(30) })
    - `createPromoCodeSchema`: z.object({ code: z.string().min(3).max(30), description: z.string().optional(), premiumDays: z.number().int().min(1), maxRedemptions: z.number().int().min(1).optional(), expiresAt: z.string().datetime().optional() })
    - Export types: RedeemCodeInput, CreatePromoCodeInput
    - Add exports to packages/shared/src/index.ts

    **3. Promo Service** -- Create `apps/api/src/services/promo.service.ts`:
    - `createCode(db, input)`: Insert into promoCodes with crypto.randomUUID() id, code uppercased
    - `listCodes(db)`: Select all promo codes ordered by createdAt desc
    - `deactivateCode(db, codeId)`: Set isActive = false
    - `redeemCode(db, userId, code)`: Full transactional redemption per RESEARCH.md pattern:
      1. Find active code (case-insensitive via .toUpperCase().trim())
      2. Check not expired, not max redeemed
      3. Check user hasn't already redeemed (unique constraint enforces too)
      4. Insert promoRedemptions record
      5. Increment currentRedemptions
      6. Extend premium: if user is premium and premiumExpiresAt > now, add days to existing expiry; otherwise add days from now
      7. Update user isPremium=true and premiumExpiresAt

    **4. Premium Service Guard** -- Modify `apps/api/src/services/premium.service.ts`:
    - In handleWebhookEvent EXPIRATION case: before setting isPremium=false, check if user.premiumExpiresAt > now (could have promo-granted time remaining). If premiumExpiresAt is still in the future, do NOT clear premium.

    **5. Routes** -- Create `apps/api/src/routes/promo.ts`:
    - `POST /promo/redeem` (authenticated): Parse body with redeemCodeSchema, call redeemCode service, return { premiumDays, newExpiresAt } or 400/409 error
    - `POST /admin/promo` (admin): Parse body with createPromoCodeSchema, call createCode, return 201 with created code
    - `GET /admin/promo` (admin): Call listCodes, return list
    - `PATCH /admin/promo/:id/deactivate` (admin): Call deactivateCode, return 200
    - Register routes in app.ts

    **6. Tests** -- Write tests FIRST in `apps/api/__tests__/services/promo.service.test.ts`. Use the existing test pattern (buildTestApp, test database). Test all behaviors listed above. If FK constraints are an issue (per known blocker), create minimal seed data (user record) in beforeEach.

    IMPORTANT: Use db.transaction() for redeemCode to prevent race conditions. The unique index on (userId, promoCodeId) is the safety net.
    IMPORTANT: Code is always stored and compared in uppercase for case-insensitive matching.
