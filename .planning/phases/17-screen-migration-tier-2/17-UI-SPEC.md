## Spacing Scale

Identical to Phase 16. All values sourced from `packages/shared/src/tokens/spacing.ts`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline chip padding, badge padding vertical |
| sm | 8px | Internal card padding, compact list row gaps, badge padding horizontal |
| md | 16px | Default element spacing, card padding, input horizontal padding |
| lg | 24px | Section padding, card vertical padding |
| xl | 32px | Layout gaps between major sections |
| xxl | 48px | Major section breaks |
| 3xl | 64px | Page-level top spacing (below collapsible header) |

Exceptions:
- Touch targets for all tappable elements: minimum 44px height (accessibility floor)
- Gold left-border accent on PostCard: 3px width (not a spacing token — fixed visual value)
- Gold top-border accent on PaywallCard: 2px height (not a spacing token — fixed visual value)
- Pull-to-refresh tint: color-only config (`tintColor="#f0c040"`)

Source: `packages/shared/src/tokens/spacing.ts`, Phase 16 UI-SPEC (inherited)

## Typography

Identical to Phase 16. All values sourced from `packages/shared/src/tokens/typography.ts`. Inter font family throughout. Two weights only: 400 (Regular) and 700 (Bold).

| Role | Size | Weight | Line Height | Token Preset |
|------|------|--------|-------------|--------------|
| Heading | 28px | 700 (Bold) | 34px (1.21) | `typography.heading` |
| Subheading | 20px | 700 (Bold) | 26px (1.30) | `typography.subheading` |
| Body | 16px | 400 (Regular) | 22px (1.375) | `typography.body` |
| Label | 13px | 400 (Regular) | 18px (1.38) | `typography.label` |

Usage rules:
- **Heading**: Screen titles, section headers, EmptyState main title
- **Subheading**: Card section titles, deck card titles, tier list names, modal sheet titles
- **Body**: List item content, description text, stat values (win rate, usage rate), toast messages, button text
- **Label**: Secondary metadata, timestamps, stat labels (e.g. "Win Rate" above the value), badge text, filter chip text, member-since field, rank number in circular badge

Text component default color: `colors.onSurface` (`#ffffff`).
Secondary/muted text: `colors.onSurfaceSecondary` (`#a0a0b8`) or `colors.onSurfaceMuted` (`#6c6c80`).

Source: `packages/shared/src/tokens/typography.ts`, Phase 16 UI-SPEC (inherited)

### 60/30/10 Distribution

| Role | Value | Token | Usage |
|------|-------|-------|-------|

| Destructive | `#e74c3c` | `palette.red[500]` | `colors.error` — Logout button, error toast, error badge |

### Accent Reserved For (Phase 17 additions in bold)

The gold accent (`#f0c040`) is used ONLY for:
1. Primary Button fill background
2. Input focus ring border
3. Pull-to-refresh tint (`tintColor` / `colors` prop on RefreshControl)
4. Custom tab bar active pill indicator (Phase 14)
5. **PostCard premium left-border accent: 3px vertical strip, linear gradient from `palette.gold[400]` to `palette.gold[600]` (D-01)**
6. **Badge `premium` variant background: `palette.gold[500]` solid fill with dark text `#0c0c18` (D-02, D-23)**
7. **PaywallCard top-border accent: 2px horizontal strip, linear gradient from `palette.gold[400]` to `palette.gold[600]` (D-24)**
8. **Profile glassmorphism header gold overlay: `rgba(240, 192, 64, 0.08)` over BlurView (D-19)**

The gold accent is NOT used for: body text, card borders, dividers, icon fills in idle state, list backgrounds, the FAB (which uses `primary` Button variant which is already gold).

## New Skeleton Compositions (D-06, D-15, D-26, D-27)

All use Phase 15 shimmer primitives: `Shimmer`, `ShimmerBox`, `ShimmerCircle`, `ShimmerText`.
Shimmer timing: `TIMING_SHIMMER` (1200ms linear — unchanged from Phase 15).

### MarketPostSkeleton

Horizontal card shape matching PostCard layout:
- Outer: Card container, full width, height 88px
- Left: ShimmerBox, 72px × 72px (image placeholder), `borderRadius.md` (12px)
- Right column: Two ShimmerText lines — 70% width (title), 45% width (metadata)
- Gap between image and text block: `spacing.sm` (8px)
- Three rows rendered, separated by `spacing.sm` (8px) gaps

### DeckRankingSkeleton

Three rows, each row:
- ShimmerCircle, 32px diameter (rank number badge)
- ShimmerBox, 40px × 40px (deck icon/avatar placeholder)
- Right column: ShimmerText 60% width (deck name) + two ShimmerBox 40px × 16px side by side (stat boxes)
- Row height: 72px
- Row padding: `spacing.md` (16px) horizontal
- Separator: Divider between rows

### TierListSkeleton

Three rows, each row:
- ShimmerText 55% width (tier list name)
- Row of four ShimmerBox 28px × 20px with `spacing.xs` (4px) gaps (tier pill placeholders)
- ShimmerText 35% width (footer metadata)
- Row height: 88px
- Row padding: `spacing.md` (16px) horizontal
- Separator: Divider between rows

### ProfileHeaderSkeleton

Single block at top of Profile screen:
- ShimmerCircle, 80px diameter (avatar)
- ShimmerText, 50% width, centered (display name)
- ShimmerBox, 60px × 20px, centered (premium badge placeholder)
- Total height: 160px
- Vertical spacing between elements: `spacing.sm` (8px)

Source: CONTEXT.md D-06, D-15, D-26, D-27 + Claude's discretion for dimensions

## Gradient Accent Patterns

### PostCard Premium Left-Border (D-01)

React Native does not support CSS `border-left` gradients natively. Implementation approach:

- Render a 3px-wide `View` on the left edge of the Card using absolute positioning
- Fill with a `LinearGradient` from `react-native-svg` (already installed — Phase 15)
- Gradient direction: vertical (top to bottom)
- Color stops: `palette.gold[400]` (`#f5d060`) → `palette.gold[600]` (`#c9a020`)
- Height: 100% of card height
- Left: 0, Top: 0, Bottom: 0
- Border radius left side: `borderRadius.lg` (16px) top-left + bottom-left only

Only shown when `post.user.isPremium === true`.

### PaywallCard Top-Border (D-24)

Same LinearGradient approach, horizontal:
- 2px-tall `View` at top edge of Card, absolute positioned
- Gradient direction: horizontal (left to right)
- Color stops: `palette.gold[400]` (`#f5d060`) → `palette.gold[600]` (`#c9a020`)
- Width: 100% of card width
- Top: 0, Left: 0, Right: 0
- Border radius top side: `borderRadius.lg` (16px) top-left + top-right only

## Animated Press Behavior

Identical to Phase 16 — `useAnimatedPress` on all tappable elements. (D-30)

| Element | Press scale | Haptic |
|---------|-------------|--------|
| Button (all variants) | 0.97 | `Haptics.impactAsync(ImpactFeedbackStyle.Light)` |
| Card (with onPress) | 0.97 | `Haptics.impactAsync(ImpactFeedbackStyle.Light)` |
| PostCard, DeckRankingCard, TierListCard | 0.97 | `Haptics.impactAsync(ImpactFeedbackStyle.Light)` |
| Sort toggle pills / filter chips | 0.95 | `Haptics.impactAsync(ImpactFeedbackStyle.Light)` |
| Vote button | 0.97 | `Haptics.impactAsync(ImpactFeedbackStyle.Light)` |
| EmptyState CTA button | via Button primitive | via Button primitive |

Spring preset: `SPRING_PRESS` (`{ damping: 15, stiffness: 300, mass: 0.8 }`).

## Staggered List Animation

`useStaggeredList` from Phase 15 — same configuration as Phase 16. (D-30)

| Screen | Applied To | Stagger delay |
|--------|-----------|---------------|
| Market tab | Post list items on initial load | `TIMING_STAGGER_DELAY` (50ms) |
| Meta tab (deck view) | Deck ranking rows on initial load | `TIMING_STAGGER_DELAY` (50ms) |
| Meta tab (tier list view) | Tier list cards on initial load | `TIMING_STAGGER_DELAY` (50ms) |
| Profile tab | Info card sections on initial load | `TIMING_STAGGER_DELAY` (50ms) |

Spring preset: `SPRING_CARD_APPEAR` (`{ damping: 12, stiffness: 120, mass: 1 }`).
Maximum staggered items: `MAX_STAGGER_ITEMS` (15).
Enter animation: fade from 0 to 1 + translate Y from 16px to 0.
Stagger count gated behind loaded data: `staggerCount = loading ? 0 : items.length` (Phase 16 decision).
