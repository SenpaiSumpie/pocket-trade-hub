  <files>apps/api/src/services/suggest.service.ts, apps/api/src/services/tierlist.service.ts, apps/api/src/routes/suggestions.ts, apps/api/src/routes/meta.ts, apps/api/src/routes/tierlists.ts</files>
  <action>
1. Create apps/api/src/services/suggest.service.ts:
   - Rule-based heuristic engine (NOT LLM -- per locked user decision)
   - export async function computeSuggestions(db, redis, userId):
     a. Check Redis cache first: key `suggestions:${userId}`, return if exists
     b. Gather signals: getUserWantedCards (from userWantedCards table), getUserCollection (from userCollectionItems), call existing getTradePower from analytics.service, call existing getAnalytics for trending data
     c. Build suggestion pairs: for each wanted card, find cards in user's collection that other users want (high trade power). Score each pair based on: trade power value, trending demand boost, high-priority wanted boost, rarity tier matching
     d. Template-generate reasoning text per pair using signal data: e.g. "{cardName} demand up {X}% this week. {N} traders are offering it." / "This is on your high priority list." / "Good value trade based on current demand."
     e. Sort by score desc, take top 5 (or fewer if not enough quality matches)
     f. Hydrate card details (name, rarity, imageUrl) for both give and get cards
     g. Cache result in Redis with 1-hour TTL (EX 3600)
     h. Store in tradeSuggestions table for history
     i. Return suggestions array matching suggestionsResponseSchema
   - Limit candidate pools: top 20 trending cards, top 20 trade power cards to avoid timeout
   - If data is sparse (user has < 3 collection items or 0 wanted), return empty suggestions with a "not_enough_data" flag

2. Create apps/api/src/services/tierlist.service.ts:
   - export async function createTierList(db, userId, data: CreateTierList): create new tier list with nanoid, validate tiers structure has at least 1 deck in any tier
   - export async function getTierLists(db, { sort, page, limit, userId? }): paginated list, sort by 'most_liked' (upvoteCount desc) or 'newest' (createdAt desc). If userId provided, also return userVoted boolean per list
   - export async function getTierListById(db, id, userId?): single tier list with userVoted
   - export async function deleteTierList(db, id, userId): only allow owner to delete, use WHERE userId = userId
   - export async function voteTierList(db, tierListId, userId): toggle vote -- if vote exists, delete it and decrement upvoteCount; if not, insert and increment. Use atomic SQL UPDATE for count (per research pitfall avoidance)
   - export async function generateOfficialTierList(db): read deckMeta sorted by winRate, auto-assign to tiers: top 10% -> S, next 20% -> A, next 30% -> B, next 25% -> C, bottom 15% -> D. Upsert the official tier list (isOfficial=true) with generated tiers

3. Create apps/api/src/routes/suggestions.ts:
   - GET /suggestions: preHandler [fastify.authenticate], check isPremiumUser. If premium: call computeSuggestions. If not premium: return { suggestions: [], isPremium: false } (mobile shows blurred teaser)
   - GET /suggestions?refresh=true: invalidate Redis cache before computing (for pull-to-refresh)

4. Create apps/api/src/routes/meta.ts:
   - GET /meta/decks: preHandler [fastify.authenticate], get isPremium, call getDeckMeta(db, isPremium). Include scrapedAt timestamp in response.
   - GET /meta/decks/:id: preHandler [fastify.authenticate], get isPremium, call getDeckDetail(db, deckId, isPremium). 404 if not found.

5. Create apps/api/src/routes/tierlists.ts:
   - GET /tierlists: preHandler [fastify.authenticate], query params: sort (most_liked|newest), page, limit. Call getTierLists with userId for userVoted.
   - GET /tierlists/:id: preHandler [fastify.authenticate], call getTierListById.
   - POST /tierlists: preHandler [fastify.authenticate], validate body with createTierListSchema. Call createTierList. Return 201.
   - DELETE /tierlists/:id: preHandler [fastify.authenticate], call deleteTierList (returns 403 if not owner, 404 if not found).
   - POST /tierlists/:id/vote: preHandler [fastify.authenticate], call voteTierList. Return updated upvoteCount and userVoted.

Follow existing route registration pattern (export default async function, prefix option).
