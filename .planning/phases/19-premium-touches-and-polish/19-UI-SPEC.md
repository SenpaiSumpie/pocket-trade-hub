| App name fade-in | opacity 0 → 1 after logo reaches 1.0, `TIMING_FADE_IN` (200ms, ease-out-quad) | Claude's Discretion — springs.ts |

### POL-02: Card Grid Layout Modes

| Mode | Columns | Card Content | Item Height |
|------|---------|-------------|-------------|
| Grid (default) | 3 | Card art only, no text overlay | ~140px (art fills cell, consistent aspect ratio) |
| Compact | 2 | Card art + name (14px Medium) + set label (13px Regular, onSurfaceMuted) | ~200px |
| List | 1 (full width) | Art thumbnail (60px wide) + name + set + rarity badge + price label in horizontal row | 76px fixed row height |

Toggle icon map (Phosphor icons, weight "Regular" default, "Bold" for active mode):
- Grid mode: `SquaresFour` icon
- Compact mode: `GridFour` icon
- List mode: `List` icon

Toggle behavior: Tap cycles grid → compact → list → grid. No popover — single tap cycles. Haptic: `haptics.navigation()` on each toggle.

Active mode indicator: accent gold (`#f0c040`) icon tint. Inactive modes: `onSurfaceMuted` (`#6c6c80`) icon tint.

Transition on mode switch: Instant (no cross-fade animation). FlashList remounts via `key={layoutMode}`. Stagger count = 0 on remount (no stagger animation on layout switch — only on initial data load).

Persistence: Zustand store with AsyncStorage via persist middleware. Store key: `layout-preference`. Default: `'grid'`.

Source: CONTEXT.md D-05, D-06, D-07, D-08; RESEARCH.md Pattern 2 (FlashList key change, store pattern)

### POL-04: Contextual Haptic Patterns

| Haptic Level | Expo API | Trigger Contexts |
|-------------|----------|-----------------|
| Navigation (Light) | `impactAsync(ImpactFeedbackStyle.Light)` | Tab switch, card tap, button press, layout toggle, scroll-to-top interactions |
| Success (Medium) | `impactAsync(ImpactFeedbackStyle.Medium)` | Trade accepted, card added to collection, proposal sent, form submitted successfully |
| Error (Heavy) | `impactAsync(ImpactFeedbackStyle.Heavy)` | Validation failure, network error, rejected action, toast error display |
| Destructive (Notification Error) | `notificationAsync(NotificationFeedbackType.Error)` | Remove from collection, cancel trade, delete post |

Hook: `useHaptics()` → returns `{ navigation, success, error, destructive }` named methods.
Worklet callers: use exported `hapticPatterns` object with `runOnJS()` wrapper (matches existing `useAnimatedPress.ts` pattern).

Reduced-motion: haptics remain active regardless of reduced-motion setting (CONTEXT.md D-17). No change to haptic behavior.

Migration: 3 existing call sites converted to `hapticPatterns.navigation()`:
1. `useAnimatedPress.ts` (line 12)
2. `CardThumbnail.tsx` (line 126)
3. `CustomTabBar.tsx` (line 72)

Source: CONTEXT.md D-13, D-14, D-15; RESEARCH.md Pattern 4

### POL-05: Reduced-Motion Accessibility

Audit scope for Phase 19 additions:
- `SplashOverlay.tsx` — instant logo + 100ms hold + fade (no spring, no shimmer)
- `useParallaxHeader.ts` — disable `translateY`, keep opacity interpolation
- `useLayoutPreferenceStore` layout switch — no animation to gate (instant remount)

Already covered (5 existing hooks, confirmed in Phase 15):
- `useAnimatedPress.ts`, `useShimmer.ts`, `useCardFlip.ts`, `useCardTilt.ts`, `AnimatedCounter.tsx`

Pattern for all new animations:
```
const reducedMotion = useReducedMotion();
if (reducedMotion) {
  sharedValue.value = targetValue; // instant set
} else {
  sharedValue.value = withSpring(targetValue, PRESET);
}
```

Source: CONTEXT.md D-16, D-17, D-18; RESEARCH.md Pattern 5
