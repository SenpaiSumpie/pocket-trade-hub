    - Use `type DbInstance = any` and accept `io: Server | null` param (same as match service)
    - `createProposal(db, io, opts)`: Generate UUID, insert into tradeProposals, if parentId provided update parent status to 'countered' with WHERE status='pending' guard (throw if 0 rows affected). Create notification via createNotification (from notification service). Emit 'new-proposal' socket event to receiver room. Call push notification helper.
    - `acceptProposal(db, io, proposalId, responderId)`: UPDATE status='accepted' WHERE id=proposalId AND status='pending' AND receiverId=responderId. If 0 rows, throw 409 error. Create notification, emit 'proposal-accepted', send push.
    - `rejectProposal(db, io, proposalId, responderId)`: Same pattern as accept but status='rejected'. Create notification, emit, push.
    - `counterProposal(db, io, opts)`: Calls createProposal with parentId set. The parent marking is handled inside createProposal. Emit 'proposal-countered'.
    - `completeProposal(db, io, proposalId, userId)`: UPDATE status='completed' WHERE id=proposalId AND status='accepted' AND (senderId=userId OR receiverId=userId). Create notification for other party, emit 'trade-completed'.
    - `getProposals(db, userId, { direction, status, limit, offset })`: Query tradeProposals. direction='incoming' -> receiverId=userId, 'outgoing' -> senderId=userId, 'all' -> either. Filter by status if not 'all'. Order by createdAt DESC. Return { proposals, total }.
    - `getProposalThread(db, proposalId)`: Find the root proposal (walk up parentId chain or query by matchId). Return all proposals in the thread ordered by createdAt ASC.
