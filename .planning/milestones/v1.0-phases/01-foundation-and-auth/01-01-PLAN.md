  <behavior>
    - POST /auth/signup: valid email+password returns 201 + {accessToken, refreshToken, user}; duplicate email returns 409; invalid input returns 400 with field errors
    - POST /auth/login: valid credentials return 200 + {accessToken, refreshToken, user}; wrong password returns 401; unknown email returns 401 (same message to prevent enumeration)
    - POST /auth/refresh: valid refresh token returns 200 + new {accessToken, refreshToken} (rotation); invalid/expired/revoked token returns 401
    - POST /auth/logout: with valid access token, revokes refresh token, returns 200; without token returns 401
    - POST /auth/reset-request: valid email returns 200 (always, even if email not found -- prevents enumeration); generates reset token stored as hash in DB
    - POST /auth/reset-confirm: valid token + new password returns 200, updates password hash, marks token used; expired/used token returns 400
  </behavior>
  <action>
    Write tests FIRST (RED), then implement (GREEN).

    **Service layer (auth.service.ts):**
    - `createUser(email, password)`: hash password with bcrypt (10 rounds), generate UUID, insert user, return user
    - `verifyCredentials(email, password)`: find user by email, bcrypt.compare, throw if invalid
    - `issueTokens(fastify, userId)`: sign access token (15m, {sub: userId}), sign refresh token (30d, {sub: userId, type: 'refresh'}), hash refresh token, store in refreshTokens table, return both
    - `refreshAccessToken(fastify, refreshToken)`: verify JWT, find token hash in DB, check not revoked/expired, revoke old token, issue new pair (rotation)
    - `revokeRefreshToken(tokenHash)`: set revokedAt on refresh token row
    - `requestPasswordReset(email)`: find user, generate random token (crypto.randomBytes), hash it, store in passwordResetTokens with 30min expiry, return raw token (for email)
    - `confirmPasswordReset(token, newPassword)`: hash token, find matching unexpired/unused row, update user password, mark token used

    **Route handlers (auth.ts):**
    - Validate request body with shared zod schemas (safeParse, return 400 on error)
    - Call service methods
    - For reset-request: log the token to console in development (skip actual email sending for now -- email integration can be added later with Nodemailer/Resend). Always return 200 regardless of email existence.
    - Return appropriate status codes and JSON responses

    **Important:** Use bcrypt for password hashing (not argon2). Store refresh tokens as bcrypt hashes. Use crypto.randomUUID() for IDs. Use crypto.randomBytes(32).toString('hex') for reset tokens, store SHA-256 hash in DB.
