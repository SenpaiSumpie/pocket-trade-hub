  <behavior>
    - registerPushToken stores token linked to userId, upserts if same userId
    - registerPushToken handles duplicate tokens gracefully
    - sendNewSetNotification fetches all push tokens and sends via Expo Push SDK
    - sendNewSetNotification handles empty token list (no-op)
    - POST /notifications/register-token requires auth, stores token
    - POST /notifications/register-token rejects invalid body
  </behavior>
  <action>
    1. Add `pushTokens` table to `apps/api/src/db/schema.ts`:
       - id (text PK), userId (text FK to users), token (text unique), platform (varchar 10), createdAt (timestamp)
    2. Create `apps/api/src/services/notification.service.ts`:
       - `registerPushToken(db, userId, token, platform)` — upsert push token (delete old token for same userId first, then insert)
       - `sendNewSetNotification(db, setName, cardCount)` — fetch all push tokens, construct Expo push messages with title "New Set Available!" and body "[SetName] -- [cardCount] new cards added", send via expo-server-sdk Expo.chunkPushNotifications + sendPushNotificationsAsync. Handle DeviceNotRegistered by deleting stale tokens.
    3. Create `apps/api/src/routes/notifications.ts`:
       - POST /notifications/register-token — preHandler: fastify.authenticate, validates body { token: string, platform: 'ios' | 'android' | 'web' } with zod, calls registerPushToken
    4. Update `apps/api/src/routes/admin.ts`:
       - After successful importCardSet, call `sendNewSetNotification(fastify.db, result.setName, result.cardCount)`
       - Import sendNewSetNotification from notification service
    5. Register notification routes in `apps/api/src/server.ts`
    6. Update `apps/api/__tests__/setup.ts` to include pushTokens in cleanDb TRUNCATE and register notification routes
    7. Create `apps/api/__tests__/services/notification.service.test.ts` — test registerPushToken (create, upsert), test sendNewSetNotification with mocked Expo SDK
  </action>
  <verify>
    <automated>cd apps/api && pnpm test -- --testPathPattern=notification -x 2>/dev/null || echo "Tests written, need DB to run"</automated>
  </verify>
  <done>Push tokens stored in DB. Notification service sends via Expo Push SDK. Token registration endpoint requires auth. Admin import triggers push notification to all users.</done>
