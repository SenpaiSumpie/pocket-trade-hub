    - bulkUpdateCollection: accepts additions (string[]) and removals (string[]), runs in transaction, additions insert with qty=1 (upsert), removals delete
    - getCollectionProgress: returns array of { setId, setName, owned, total } computed with single SQL query joining cards and userCollectionItems grouped by setId
    - getUserCollection: returns all collection items for a user (cardId + quantity), optionally filtered by setId
    - addToWanted: inserts with default priority 'medium'; on duplicate upserts
    - removeFromWanted: deletes row by userId + cardId
    - updatePriority: sets priority to high/medium/low
    - getUserWanted: returns all wanted items for a user (cardId + priority), optionally filtered by setId
  </behavior>

       - Follow exact pattern from card.service.ts (type Db = any, pure functions)
       - `addToCollection(db, userId, cardId, quantity=1)`: upsert using onConflictDoUpdate on (userId, cardId) unique index, incrementing quantity

       - `updateQuantity(db, userId, cardId, quantity)`: if quantity <= 0, call removeFromCollection; else update quantity and updatedAt
       - `bulkUpdateCollection(db, userId, additions, removals)`: wrap in db.transaction, batch insert additions (upsert, qty=1) in groups of 50, batch delete removals
       - `getCollectionProgress(db, userId)`: single query joining cards + sets with LEFT JOIN userCollectionItems, grouped by setId, returning { setId, setName, owned, total }

       - `removeFromWanted(db, userId, cardId)`: delete where userId AND cardId
       - `updatePriority(db, userId, cardId, priority)`: update priority and updatedAt
       - `getUserWanted(db, userId, setId?)`: select all wanted items for user, optionally filtered by setId
