  <files>apps/mobile/src/hooks/useImageExport.ts, apps/mobile/src/components/export/ExportRenderer.tsx, apps/mobile/src/components/export/ShareButton.tsx, apps/mobile/src/components/export/templates/CollectionExport.tsx, apps/mobile/src/components/export/templates/PostExport.tsx, apps/mobile/src/components/export/templates/WantedExport.tsx, apps/mobile/src/components/export/templates/CardExport.tsx</files>
  <action>
    **0. Install dependencies:**
    ```bash
    cd apps/mobile && npx expo install react-native-view-shot expo-sharing
    ```

    **1. useImageExport hook** -- Create `apps/mobile/src/hooks/useImageExport.ts`:
    - Returns `{ viewRef, exportAndShare, exporting }` where exporting is a boolean loading state
    - `exportAndShare`: calls captureRef(viewRef.current, { format: 'png', quality: 1, result: 'tmpfile' }), then Sharing.shareAsync('file://' + uri, { mimeType: 'image/png', dialogTitle })
    - IMPORTANT: Always prepend 'file://' to captureRef result for Android compatibility
    - Handle errors gracefully (Alert.alert on failure)

    **2. ExportRenderer** -- Create `apps/mobile/src/components/export/ExportRenderer.tsx`:
    - Wrapper component that renders children offscreen for capture
    - Uses `position: 'absolute', left: -9999` (NOT display: 'none') so React Native actually renders it
    - Sets `collapsable={false}` on the inner View (required for captureRef on Android)
    - Accepts `ref` via forwardRef for the capture target
    - Fixed width (e.g., 1080px) for consistent export resolution regardless of device

    **3. ShareButton** -- Create `apps/mobile/src/components/export/ShareButton.tsx`:
    - Reusable TouchableOpacity with share icon (use Ionicons "share-outline")
    - Props: `onPress`, `loading` (shows ActivityIndicator when exporting)
    - Styling: subtle icon button, app theme consistent

    **4. Four export templates** in `apps/mobile/src/components/export/templates/`:

    **CollectionExport.tsx:**
    - Props: setName, completionPercent, cardImages (uri[]), totalCards, ownedCards, showWatermark
    - Layout: Dark background (#1a1a2e or app dark bg), gold accent header with set name and completion %, card thumbnail grid (up to 20 cards), stats overlay
    - forwardRef for capture target

    **PostExport.tsx:**
    - Props: postType ('offering'|'seeking'), cards (name, image, rarity)[], posterName, showWatermark
    - Layout: Dark background, post type badge (Offering/Seeking), card list with thumbnails and names, poster attribution
    - forwardRef for capture target

    **WantedExport.tsx:**
    - Props: wantedCards (name, image, rarity, priority)[], userName, showWatermark
    - Layout: Dark background, "Wanted Cards" header, card grid with priority indicators, user name
    - forwardRef for capture target

    **CardExport.tsx:**
    - Props: cardName, cardImage, rarity, setName, showWatermark
    - Layout: Dark background, large card image centered, card name, rarity badge, set name below
    - forwardRef for capture target

    **All templates common:**
    - Dark background consistent with app theme
    - Gold accent (#f0c040) for headers and highlights
    - `showWatermark` prop: when true, render "Pocket Trade Hub" text watermark (semi-transparent, bottom-right or diagonal)
    - `collapsable={false}` on root View
    - Fixed dimensions for consistent export (e.g., 1080x1920 or 1080x1350 depending on content)
  </action>
