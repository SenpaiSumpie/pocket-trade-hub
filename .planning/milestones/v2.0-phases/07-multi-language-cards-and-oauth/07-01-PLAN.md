<task type="auto">
  <name>Task 1: Add DB schema tables and alter existing tables</name>
  <files>apps/api/src/db/schema.ts</files>
  <action>
Modify the existing Drizzle schema file to add:

1. **Shared language constant** at the top of the file:
```typescript
export const supportedLanguages = ['en', 'de', 'es', 'fr', 'it', 'ja', 'ko', 'pt', 'zh'] as const;
export type CardLanguage = typeof supportedLanguages[number];
```

2. **New `cardTranslations` table** after the `cards` table:
- `id` TEXT PK (format: `{cardId}-{lang}`)
- `cardId` TEXT NOT NULL references cards.id
- `language` VARCHAR(5) NOT NULL
- `name` VARCHAR(100) NOT NULL
- `imageUrl` TEXT NOT NULL
- `attacks` JSONB (nullable, translated attack names/descriptions)
- `createdAt` TIMESTAMP DEFAULT NOW NOT NULL
- Unique index on (cardId, language)
- Index on language
- Index on lower(name)

3. **New `oauthAccounts` table** after refreshTokens:
- `id` TEXT PK
- `userId` TEXT NOT NULL references users.id
- `provider` VARCHAR(20) NOT NULL ('google' | 'apple')
- `providerUserId` TEXT NOT NULL
- `email` VARCHAR(255) nullable
- `createdAt` TIMESTAMP DEFAULT NOW NOT NULL
- Unique index on (provider, providerUserId)
- Index on userId

4. **Alter `users` table:**
- Change `passwordHash` from `.notNull()` to no notNull (make it nullable)
- Add `preferredCardLanguage` VARCHAR(5) DEFAULT 'en'

5. **Alter `userCollectionItems` table:**
- Add `language` VARCHAR(5) NOT NULL DEFAULT 'en'
- Change unique index from `(userId, cardId)` to `(userId, cardId, language)`

6. **Alter `userWantedCards` table:**
- Add `language` VARCHAR(5) NOT NULL DEFAULT 'en'
- Change unique index from `(userId, cardId)` to `(userId, cardId, language)`

After modifying the schema, run `cd apps/api && npx drizzle-kit generate` to create the migration, then `cd apps/api && npx drizzle-kit push` to apply it. If drizzle-kit is not set up for migrations, use `drizzle-kit push` directly to sync schema.

IMPORTANT: The `passwordHash` column changing from NOT NULL to nullable is a breaking change for `verifyCredentials` in auth.service.ts. Do NOT modify auth.service.ts in this plan -- that happens in Plan 03. The schema change is safe because existing rows all have passwordHash values.

<task type="auto">
  <name>Task 3: Create multi-language card translation seed script</name>
  <files>apps/api/src/db/seeds/seed-translations.ts, apps/api/package.json</files>
  <action>
Create a new seed script `seed-translations.ts` that imports card translations from TCGdex for all 6 available languages (en, de, es, fr, it, pt). Do NOT modify the existing `seed-cards.ts` -- the translations script runs AFTER the base cards are seeded.

The script should:

1. **Accept `--dry-run` flag** (same pattern as seed-cards.ts)
2. **Connect to DB** using DATABASE_URL env var
3. **Iterate over the 6 available languages** (NOT ja, ko, zh -- those return 404 from TCGdex)
4. **For each language, fetch each set** from `https://api.tcgdex.net/v2/{lang}/series/tcgp`
5. **For each card in each set**, upsert into `cardTranslations` with:
   - `id`: `{cardId}-{lang}` (e.g., `A1-001-de`)
   - `cardId`: the base card ID (e.g., `A1-001`) -- must exist in `cards` table
   - `language`: the language code
   - `name`: card name from TCGdex (translated)
   - `imageUrl`: from TCGdex response, pattern `{card.image}/high.webp`, fallback to `https://assets.tcgdex.net/{lang}/tcgp/{setId}/{localId}/high.webp`
   - `attacks`: translated attack data (same transform as seed-cards.ts)
6. **Use onConflictDoUpdate** on (cardId, language) to allow re-running safely
7. **Add 500ms delay** between API calls (same as existing seed script)
8. **Skip cards not found in the base `cards` table** (some languages may have fewer cards)
9. **Log progress**: language, set name, card count per set, total per language
10. **Handle errors gracefully**: if a language endpoint fails, skip that language and continue

Add an npm script in `apps/api/package.json`: `"seed:translations": "tsx src/db/seeds/seed-translations.ts"`

Note: English translations should also be seeded into cardTranslations (using data from the existing cards table or re-fetching from TCGdex EN). This ensures the translations table is the single source for localized card data.
  </action>
