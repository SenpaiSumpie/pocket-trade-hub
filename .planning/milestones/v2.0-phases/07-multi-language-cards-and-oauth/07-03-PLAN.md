```typescript
  // - Parse body with oauthLoginSchema (provider must be 'apple')
  // - Call verifyAppleToken(idToken)
  // - Call findOrCreateOAuthUser(db, 'apple', payload.sub, payload.email)
  // - If needsLinking, return 200 with { needsLinking: true, email, provider: 'apple' }
  // - Otherwise, call issueTokens and return 200 with tokens + user

  // POST /auth/link (requires authentication)
  // - preHandler: [fastify.authenticate]
  // - Parse body with linkAccountSchema
  // - Verify the idToken with the appropriate provider
  // - Call linkOAuthAccount(db, request.user.sub, body.password, provider, providerUserId, providerEmail)
  // - Return 200 with { message: 'Account linked successfully' }
  // - On password error, return 401
  // - On duplicate provider link, return 409
}
```

Response format for OAuth login endpoints:
- Success (new or existing user): `{ accessToken, refreshToken, user: { id, email, displayName, ... } }`
- Needs linking: `{ needsLinking: true, email: "user@example.com", provider: "google" }`
- Invalid token: 401 `{ error: "Invalid token" }`

Response format for link endpoint:
- Success: 200 `{ message: "Account linked successfully" }`
- Wrong password: 401 `{ error: "Invalid password" }`
- Already linked: 409 `{ error: "Provider already linked to this account" }`
